# Playbook: Employee Offboarding

Internal reference only — grounding for `workflow-def`'s best-practice suggestions. Never shown verbatim to the interviewee; use it to recognize which canonical step a described action maps to, and to notice gaps.

## Canonical steps

1. Resignation/termination decision recorded, last working day confirmed.
2. Manager and HR notified; offboarding checklist opened.
3. Knowledge-transfer plan created (docs, handover meetings, ownership reassignment).
4. Access review: list every system/tool the person can reach.
5. Scheduled access revocation timed to the last working day (not before, not late).
6. Hardware/asset return process initiated (laptop, badge, keys, phone).
7. Final payroll and benefits/COBRA-equivalent handled.
8. Exit interview conducted and notes recorded.
9. Internal announcement to team/stakeholders about the departure and handover owner.
10. Redirect or archive the person's email/inbox and shared-file ownership.
11. Remove from distribution lists, shared calendars, and recurring meetings.
12. Confirm no orphaned service accounts or API keys remain tied to the person.

## Commonly missed steps

- Access revocation not tied to a specific timestamp — accounts linger active for days/weeks.
- No knowledge-transfer step before the last day; institutional knowledge walks out the door.
- Orphaned integrations/API keys/service accounts owned by the departing person never rotated.
- Shared file/drive ownership not reassigned, leaving orphaned resources.
- No exit interview captured, losing a feedback signal.

## Needed documents

- Offboarding checklist
- Exit interview template/questionnaire
- Asset return confirmation form
- Access revocation checklist (systems inventory)
- Final pay / benefits settlement documentation
- Knowledge-transfer handover doc template
- Non-compete / confidentiality reminder letter
